# 你好，博客世界

> 种一棵树最好的时间是十年前，其次是现在。

很早就想拥有一个自己的技术博客，用来沉淀学习的过程，分享踩坑的经验。今天终于动手了。

## 为什么选择 VitePress

市面上博客方案很多——Hexo、Hugo、Next.js、Ghost……最终我选择了 [VitePress](https://vitepress.dev/)，理由如下：

- **零依赖的 Markdown 写作体验** —— 直接写 `.md` 文件，没有任何心理负担
- **Vue 生态无缝集成** —— 可以在 Markdown 里写 Vue 组件
- **Vite 极速构建** —— 热更新快如闪电，写作反馈即时
- **GitHub Pages 免费托管** —— push 代码即发布，省心省力

## 博客的定位

这不是一个只写前端的博客。这里会记录我在技术世界里探索的一切——

1. **系统设计** —— 架构思考、高可用、分布式的每一个有趣问题
2. **后端开发** —— Go / Python / Node.js 的语言特性与实战经验
3. **前端工程** —— 现代框架、工具链、性能调优
4. **AI 与大模型** —— 应用落地、提示工程、工具链探索
5. **效率工具** —— 开发环境、自动化、让日常工作更顺滑的一切
6. **学习笔记** —— 读书摘要、课程精华、技术文档的深度解读

技术本来就是一张网，只盯着某一个节点，会错过很多有趣的连接。

## 一个小目标

> **坚持写作，每月至少 2 篇。**

欢迎通过 GitHub Discussions 留言交流，或者直接提 Issue。
